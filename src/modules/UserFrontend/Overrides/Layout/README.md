# Layout Overrides - User Frontend Examples

Ця папка містить приклади використання системи перевизначень лейаутів через PHP атрибути.

## Як це працює

Система дозволяє модифікувати лейаути модулів без зміни їх XML файлів. Всі зміни оголошуються в класах з атрибутом `AsLayoutOverride` в папці `src/`.

## Приклад: LoginLayoutOverride

Файл `LoginLayoutOverride.php` демонструє:

1. **ADD операцію** - додавання нових блоків у лейаут
2. **Позиціювання** - використання `before` та `after` для контролю порядку блоків
3. **Передачу аргументів** - передача даних у блоки через `args`

### Структура операцій

```php
#[AsLayoutOverride(
    handle: 'login',  // Назва layout handle
    operations: [
        [
            'type' => 'add',           // Тип операції: 'add', 'remove', 'move'
            'into' => 'main',          // Контейнер, куди додати блок
            'before' => 'BlockName',   // Вставити перед блоком (опціонально)
            'after' => 'BlockName',    // Вставити після блоку (опціонально)
            'block' => [
                'name' => 'BlockName',                    // Ім'я блоку
                'template' => '@module/block/template.twig', // Шаблон
                'args' => [                               // Аргументи для шаблону
                    'key' => 'value'
                ]
            ]
        ],
        [
            'type' => 'remove',
            'name' => 'BlockToRemove'
        ],
        [
            'type' => 'move',
            'name' => 'BlockToMove',
            'into' => 'targetContainer',
            'after' => 'ReferenceBlock'
        ]
    ],
    priority: 100  // Пріоритет (вищі значення застосовуються пізніше)
)]
```

## Результат

Після застосування override, лейаут `login` буде містити:

1. **LoginPromoBlock** - промо-блок з привітанням (перед формою)
2. **LoginFormBlock** - оригінальна форма входу
3. **LoginInfoBlock** - інформаційний блок (після форми)
4. **LoginHelpBlock** - блок допомоги (після інфо-блоку)

## Переваги

- ✅ Не потрібно змінювати XML файли модулів
- ✅ Всі зміни в одному місці (`src/`)
- ✅ Легко відстежити та підтримувати
- ✅ Можливість використовувати пріоритети для складних сценаріїв
- ✅ Автоматичне застосування при завантаженні лейауту
